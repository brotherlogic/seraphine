# Seraphine

Seraphine is an agentic workflow, written largely in golang with a heavy emphasis on defined stages and SKILLs that support those stages.

It is currently an experimental workflow, so treat it is exploratory.

## Workflow

Lean on github issues as the primary means of communication and recording actions - store any plans in an associated github issue.

Always start on the main branch, and sync to head.

Features should be pushed to seperate branches.

Never create pull requests manually - they will be automatically built on branch pushes.
